# 项目总览（fm）

`fm` 是一个面向个人开发者的「项目文件夹管理器」。它把散落在硬盘各处的项目目录（游戏、Node 后端、C# 小工具、文档等）统一索引、分类、打标签，并提供优雅简洁的浏览界面。

## 核心理念

1. **配置即数据库**：所有索引、分类、标签都存放在一份 JSON 配置文件中，便于直接阅读、版本化和迁移。
2. **目录自描述优先**：项目根目录可放置 `.meta-data`（JSON）描述自身；存在则以其为准，不存在则回落到配置文件中按路径/名称匹配的记录。
3. **零侵入扫描**：通过预指定的「扫描根」+ 可控深度递归发现项目；尊重 `.gitignore` 与配置内自定义的忽略规则。
4. **为同步留口子**：忽略规则、相对路径、可移植配置——为后续「跨机同步」预留地基（M2）。

## 用户旅程（M1）

```
首启  ──►  选择/创建配置文件
   │
   ├─►  添加扫描根目录（如 D:\projects, E:\code）
   │
   ├─►  扫描   ──►  发现项目  ──►  写入数据库（含 .meta-data 合并）
   │
   ├─►  浏览：左侧分类树 / 主区项目卡片 / 右侧详情
   │
   └─►  编辑：名称、类型、描述、标签 → 选择写回 .meta-data 或仅存数据库
```

## 范围划分

| 阶段 | 内容 |
|------|------|
| **M1（当前）** | 配置文件加载/保存、扫描、列表浏览、详情查看、元数据编辑、`.meta-data` 读写 |
| M2 | 多机同步（基于忽略规则的差异传输）、配置版本迁移 |
| M3 | 全文搜索、README 摘要预览、项目操作（在 IDE 中打开、终端打开、复制路径） |

## 关键设计决策（已与用户对齐）

- **配置位置**：默认置于可执行文件同级 `fm.config.json`；用户可在界面切换/加载任意路径的配置文件。
- **元数据策略**：`.meta-data` 优先，DB 兜底。两者均存在时以 `.meta-data` 为准，DB 中相应字段视为缓存。
- **首版范围**：M1。
- **UI 风格**：Linear / Raycast 极简风——中性灰、细线分隔、克制留白、暗色支持。
