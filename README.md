# hyper-fm

`hyper-fm` 是用于管理和同步文件夹的桌面应用，主要功能：

- **集中查看项目**：集中展示散落在不同目录的项目，按标签和标签组筛选查看。
- **维护项目信息**：存储项目名称、描述、标签等信息。
- **多设备共享**：通过共享目录、P2P等同步机制实现跨设备管理。

## 使用说明

### 界面导览

![fm 界面导览](docs/assets/app-screenshot.png)

- **侧边栏**：选择标签和标签组，筛选项目。
- **主区域**：浏览项目卡片、搜索项目。
- **项目详情**：点击项目卡片查看详细信息，编辑名称、描述、标签和元数据。
- **设置面板**：管理配置文件、扫描根、忽略规则和同步设置。

### 托盘与快速操作

- 默认启用托盘：关闭主窗口后不会直接退出，应用会继续保留在系统托盘中。
- 右键托盘图标可直接看到当前项目列表，并展开二级菜单执行常用操作，例如：打开文件夹、在 VS Code 中打开、运行自定义命令。
- 托盘菜单同时提供软件级操作，例如显示主窗口、按同步配置执行快速同步、退出应用。
- 如果你不希望关闭窗口后保留后台运行，可在“软件设置 → 托盘”中关闭该功能；该偏好保存在用户目录下的 `.fm.json` 中，不会写入共享配置文件。

### 基本使用

#### 管理项目
1. 启动应用，如果还没有配置，选一个目录创建新的配置文件。
2. 点击【添加项目】添加需要管理的项目；也可以在按钮菜单中使用【批量添加项目】一次选择多个目录。
3. 在主界面搜索、筛选和浏览项目。
4. 点击项目进入详情页，编辑项目信息和标签。

#### 同步项目
1. 在同步设置面板添加同步方式，例如共享目录、P2P。
2. 将项目文件导出到文件夹或同步到云端。
3. 将 `共享配置文件` 复制到其他设备。
4. 在其他设备启动软件，导入 `共享配置文件`。
5. 添加 `扫描根目录` 识别项目或指定路径拉取项目。

### 核心概念

- **项目**：一个文件夹，根据 `文件夹名称`/`元数据文件`/`文件指纹` 唯一识别。
- **共享配置文件**：包含项目元数据的 JSON 文件，用于在设备间同步项目信息。
- **推荐本地配置目录**：工作目录下的 `.local/`，适合存放你自己的 `fm.shared.json` 与 `fm.local.json`，不会提交到 Git。
- **扫描根目录**：用于在本机查找共享配置文件中已添加的项目。
